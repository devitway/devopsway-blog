# Контент-гайд: примеры постов, аудит когерентности, терминология

## 1. Примеры постов канала @DevITWay (апрель-май 2026)

### Форматы и паттерны

| Формат | Хештеги | Длина | Пример |
|--------|---------|-------|--------|
| Кейс (продукт) | #кейс #devops | 800-1500 | Pusk на Хабре |
| Дайджест | #дайджест #security | 2000-3000 | DevOps Digest #9-13 |
| Серия-мнение | #мышление #ai | 1500-2500 | Барьер входа (6/6) |
| Карьера | #карьера #devops | 1000-1500 | Смена домена, Career Matcher |
| Мини-курс | #миникурс #linux | 800-1200 | Linux 10 принципов |
| Продуктовый кейс | #кейс #ai #mlops | 1500-2500 | MLOps серия (5/5) |

### Голос канала

- Без эмодзи в теле поста (кроме дайджестов)
- Без заголовков в TG-постах (заголовок = первая строка)
- Ссылки без markdown -- просто URL
- 2 хештега в конце (максимум 3-4 для дайджестов)
- Тон: сухой, конкретный, без прилагательных-украшений
- Хук: мысль или провокация, не число ради числа
- Ссылки на предыдущие посты создают нарратив между постами
- Русский язык, англицизмы только для устоявшихся терминов

### Структура типичного поста

```
[Хук: 1-2 предложения, факт или провокация]

[Контекст: зачем это важно, 2-3 предложения]

[Техническая суть: конкретика, числа, команды]

[CTA или мост к следующему посту]

ссылка

#тег1 #тег2
```

## 2. Примеры постов (референсы)

### Кейс: Pusk на Хабре (03.04.2026)

Формат: #кейс #devops, ~1200 зн.
Хук: ирония ("писать в Telegram о том, что Telegram заблокировали")
Структура: проблема → решение → архитектура → ссылки
CTA: статья + GitHub

### Серия: Барьер входа (6 постов, 06-11.04.2026)

Формат: #мышление #ai, 1500-2500 зн. каждый
Нумерация: (1/6)...(6/6)
Хук: цитата или провокация
Нарратив: каждый пост заканчивается "Продолжение: завтра."
Финальный пост: actionable-советы

### Серия: MLOps (5 постов, 15.04-28.04.2026)

Формат: #кейс #ai #mlops
Структура: практический нарратив (поднял → нашёл → сломалось → починил)
Ссылки на свои продукты: Code Roast, MLflow стенд, Grafana дашборд
Каждый пост заканчивается анонсом следующего

### Дайджест (#9-13)

Формат: #дайджест + тематические теги
Структура:
- Главное (5-7 пунктов с ссылками)
- Аналитическая врезка (абзац)
- Команда дня (одна команда с пояснением)
- CTA: "Переслал коллегам? Пусть подпишутся"

### Пост-мост к RAG-серии (05.05.2026)

"Общая память: почему .md -- это не RAG"
Формат: #миникурс #мышление
Аналогия: Матрица/Нео
Заканчивается: "Скоро про Qdrant. Минимум теории, только конфиги и запуск."
→ Это прямой lead-in к rag-01-qdrant-vectors.md

## 3. Аудит когерентности (чеклист)

При написании поста или тизера проверять:

### Внутренняя логика
- [ ] Каждое утверждение следует из предыдущего
- [ ] Нет противоречий между секциями
- [ ] Примеры подтверждают тезис, а не опровергают

### Обещание → доставка
- [ ] Хук/вступление обещает то, что артефакт реально даёт
- [ ] Если написано "отправил алерт" -- скрипт действительно отправляет
- [ ] Если написано "сервер сообщает" -- есть механизм нотификации

### Консистентность терминов
- [ ] Один концепт = одно слово на протяжении всего текста
- [ ] cosine similarity ≠ cosine distance (противоположные понятия!)
- [ ] Новый термин объяснён при первом использовании
- [ ] Английский термин при первом использовании, дальше русский

### Кросс-текстовая связность
- [ ] Тизер не обещает того, чего нет в посте
- [ ] Числа в тизере совпадают с числами в посте
- [ ] CTA ведёт на правильный URL

### Найденные баги (май 2026)

| Баг | Где | Суть |
|-----|-----|------|
| cosine distance vs similarity | Мини-тест Qdrant | distance и similarity -- противоположные меры, перепутаны |
| Hook → артефакт | Linux automation | "Отправил алерт в Telegram" -- скрипт не отправляет, только логирует |
| Дублирование | Qdrant пост | Аргумент про LIKE '%proxy%' повторён дважды |
| Термин без введения | Qdrant мини-тест | "чанк" использован до объяснения |
| "сравнивает значения" | Qdrant тизер | "значения" двусмысленно, правильно "смыслы" |
| "формат знакомый" | Linux тизер | Предполагает контекст, которого у нового читателя нет |

## 4. Стандарт терминологии для блога

### Правило

Профессиональный русский с английским оригиналом в скобках при первом упоминании. В дальнейшем -- только русский вариант.

### Словарь

| Русский | English | Примечание |
|---------|---------|------------|
| Косинусная близость | cosine similarity | НЕ "косинусное расстояние" (это обратная мера) |
| Векторизация | embedding | Процесс превращения текста в вектор |
| Модель-эмбеддер | embedding model | Модель, которая делает векторизацию |
| Ранжирование | ranking | Сортировка результатов по релевантности |
| Подстановка контекста | context injection | Вставка результатов поиска в промпт |
| Оценка релевантности | score | Числовая мера похожести |
| Метаданные | metadata | Сопроводительные данные к вектору |
| Фрагмент текста (чанк) | chunk | При первом использовании -- с расшифровкой |
| Векторная БД | vector database | |
| Семантический поиск | semantic search | |

### Что НЕ переводится

- Имена из API: `"distance": "Cosine"`, `cosine()`, `upsert`
- Названия инструментов: Qdrant, Ollama, all-MiniLM
- Параметры конфигов: `size`, `distance`, `limit`
- Имена команд: `curl`, `docker run`, `systemctl`

## 5. Маппинг серии Linux: 10 принципов

| # | Принцип (KOKON) | Linux-тема | Артефакт |
|---|-----------------|------------|----------|
| 1 | Автоматизация (C01) | bash, cron, systemd timers | health-check.sh + timer |

| 7 | Безопасность (C10) | chmod, sudo, SSH, fail2ban | "Первые 10 минут" скрипт |
| 8 | Масштабирование (C04) | ulimits, sysctl, tmpfs, swap | sysctl.conf для production |
| 9 | Декларативность (C05) | systemd units, fstab, cloud-init | cloud-init конфиг |
| 10 | Оркестрация (C06) | systemd deps, Makefile, xargs | Makefile для 5 сервисов |
